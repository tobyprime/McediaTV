Part I
总览概述
在 Minecraft 里和朋友们一起同步看视频听音乐——这套系统怎么把画面、声音和播放内容分开管理。
I · Overview

MTV 是什么

MTV 让多人一起看视频听音乐，画面和声音在多台电脑上同步。
播放器就是一台电视机——屏幕、喇叭、摆放位置都在它身上。
频道就是电视台——多台电视调到同一个台，播的就是同一套内容。
日常入口 /mtv，操作管的是频道，不是电视机本身。

每块画面都属于某个播放器，每个播放器都绑定了一个频道。调画面去播放器页，改内容去频道页。多台播放器绑同一个公共频道——所有玩家看到的画面同步播放。

播放器是一台电视机，频道是一个电视台

MTV 把硬件和内容分成两层：

  • 播放器（Player）——物理层。屏幕挂哪、喇叭放哪、电源开没开，都由播放器管。屏幕和喇叭是播放器自带的部件。
  • 频道（Channel）——内容层。播什么视频、列表顺序、暂停还是播放，都由频道管。一个频道能被多个播放器绑定。
部件负责什么类比
播放器本体名字、位置、电源、总音量、激活范围电视机外壳
屏幕外设画面——尺寸、亮度、偏移、旋转、弹幕电视机屏幕
扬声器外设声音——音量、声道、出声位置电视机喇叭
频道内容——媒体、列表、暂停、进度电视台信号

默认配置

新建播放器默认带 1 块屏幕、2 个喇叭，总音量 1.0，激活范围 50 格，绑定私有频道，状态为私有。其中两个喇叭默认分别偏向左右声道。

快速定位：调画面→播放器页；改内容→频道页；多屏同播→公共频道列表；远程遥控→遥控器页。
Part II
用户操作指南
打开 GUI、创建播放器、设置频道、调整画面和声音——每个操作要进哪个页面，怎么找到对应的按钮。
II · User Guide

快速上手

打开 /mtv，点"创建播放器"，聊天框里输个名字——播放器就出现在你脚下了。

自动跳转到控制页后，打开"频道设置"，点"设置 URL"，贴个播放链接进去，画面就出来了。嫌画面不对、喇叭没声，再去"外设管理"调。

也可以直接输 /mtv create <名字> 跳过主菜单。

主菜单导航

/mtv 打开的主菜单分两块：左边管播放器，右边管频道。

播放器侧

  • 控制最近的播放器：自动找周围 50 格内最近的 MTV。
  • 附近的播放器：列出附近所有 MTV，显示名称、距离和绑定频道。
  • 创建播放器：在当前脚下位置新建一个。

频道侧

  • 频道控制：操作最近播放器的频道（切歌、暂停、列表）。
  • 频道列表：浏览全部公共频道。
  • 我的频道：只看自己创建的。

播放器页面

播放器页管的是这一台机器的所有设置，从名字到绑定频道都在这里。

操作说明
重命名聊天栏输新名字。
传送到位置直接飞过去。
外设管理调所有屏幕和喇叭的详细设置。
位置与朝向微调坐标、朝向。左键减、右键加，潜行步进更大。支持吸附整数、传送到脚下、重置朝向。
激活范围设多远距离内保持播放。支持快速增减或精确输入。
公开/私有切换切换是否对其他人可见。
绑定频道绑到一个公共频道，跟其他播放器共享内容。
切回私有频道解除公共频道绑定，回到独立模式。
删除播放器永久移除播放器和所有外设，没有撤销。

屏幕与扬声器设置

"外设管理"里列出了所有的屏幕和喇叭，点进去就能细调。

屏幕设置

参数如何调整
尺寸（宽/高）点击增减，潜行步进更大。
亮度0 ~ 15 级。点按 1 级，潜行 5 级。
填充模式保持比例（留黑边）或铺满（填满屏幕）。
弹幕直接切换开关。
偏移（X/Y/Z）左键减、右键加；普通步进 0.1，潜行步进 1.0。支持重置、吸附整数、设到玩家位置。
旋转三轴独立微调，步进 5°，潜行步进 25°。支持一键重置。
小技巧：先站到你想放屏幕的位置，点"设到玩家位置"，偏移自动对齐，再微调旋转和尺寸。

屏幕页还提供"重置显示"（恢复默认显示参数）和"删除屏幕"两项操作。

扬声器设置

参数如何调整
音量每个扬声器独立可调；普通步进 0.1，潜行步进 1.0。
范围声音传播距离；普通步进 1，潜行步进 5。
声道混合 / 左 / 右。单喇叭用混合，左右各一个就分别设左/右。
偏移调整出声位置，操作跟屏幕偏移一样。

扬声器页还提供"重置音频"（恢复默认音量、范围、声道）和"删除扬声器"两项操作。

频道页操作

频道页管的是在播什么。不管是你自己的频道还是跟别人共用的，页面操作都一样。

播放列表

顶部列表一次显示 7 项，可滚动：

操作效果
左键立即播放
右键删除该项
Shift + 左键移到最前
Shift + 右键移到最后

左右箭头滚动列表。

管理按钮

  • 首加 / 下一个加 / 尾加：加到列表不同位置，需要聊天栏输入链接。
  • 清空列表：清掉所有项并停播。
  • 播放顺序：循环切换播放模式。

播放顺序

模式效果
列表播放按顺序播一遍就停。
随机打乱顺序轮播。
列表循环播完自动重头。
当前媒体循环单曲/单视频反复播。
播完当前停止播完就停。

播放控制

频道页中部是播放/暂停，底部一排从左到右：上一个、退 20 秒、退 5 秒、设置 URL、进 5 秒、进 20 秒、下一个。

注意："设置 URL"不是清空替换——它把新链接插到当前播放内容后面，然后立刻切过去。想彻底换掉，先点"清空列表"再设 URL。

遥控器

站远处随手控播，不用进菜单翻找。

功能说明
电源开关暂时关掉或重新打开（不是删除设备）。
总音量所有喇叭的总音量（不是单个喇叭的）。
上一首 / 下一首切歌。
快退 / 快进后退或前进 5 秒 / 20 秒。
设置 URL输个链接直接放。
进入设置页跳到播放器或频道设置。

典型场景与常见问题

场景一：单人私有电视

适合家园电视、小房间屏幕。创建播放器后保持默认（私有状态 + 自带频道），去频道页设个链接就能播。不改任何设置也能用。

场景二：大厅公告大屏

适合主城迎宾屏、活动预告屏。让服主建一个公共频道，把几台播放器都绑过去。都调到同一个频道后，所有屏幕播同一套内容。

场景三：公共点歌屏

适合酒吧、休闲区。频道需要开放公开控制，这样谁都可以点播。在频道页加链接或从列表里选一个就行。

常见问题

为什么我能看到频道，但点了没反应

频道没开公开控制，你也没有操作权限。能看，不能改。

为什么播放器离远了就不播了

超出激活范围了。走远会自动关，走回来自动恢复。

为什么设置 URL 后不是替换，而是插入

"设置 URL"是插到当前内容后面再切过去，不是清空重来。想彻底换掉，先点"清空列表"，再设 URL。

功能与兼容性 QA

支持哪些视频格式、网络视频和直播流？

这部分不是由 MTV 本体硬编码决定的，而是交给独立的视频平台解析 Mod来处理。你装了哪些平台的解析 Mod，就支持哪些平台的链接或分享文本。

可以把它理解成两步：先解析，再播放。服务端插件或客户端 MTV Mod 只负责把玩家输入的链接、分享文本分发出去；平台解析 Mod 再把它解析成 FFmpeg 能直接处理的真实媒体链接，然后客户端本地解码播放。

所以会出现一种情况：同一个链接，A 玩家能播，B 玩家不一定能播。原因通常不是频道坏了，而是 A 装了对应的平台解析 Mod，B 没装，或者两边装的解析能力不一样。

支持多人同步观看同一个视频吗？中途加入或重新进入观看区域时会自动同步进度吗？

支持。多个播放器绑到同一个公共频道后，会共享同一套播放状态和播放列表。玩家中途加入、重新靠近播放器，或者重新进入观看区域时，客户端会拿到当前频道状态并自动对齐到当前进度，不会各播各的。

支持创建多个独立视频屏幕和播放列表吗？

支持。你可以创建多台互不影响的播放器，每台保留自己的私有频道和播放列表；也可以让多台播放器绑定到同一个公共频道，做多屏同播。

管理员可以通过哪些方式控制视频播放？

主要有两种：GUI 和 命令。日常推荐直接用 /mtv 打开 GUI，通过播放器页、频道页、遥控器页控制；项目里也保留了旧式命令入口，更适合管理员调试、兼容或批量处理。

支持哪些服务端和客户端平台（Paper、Fabric、Forge 等）？

客户端只支持 Fabric。 服务端只支持类 Paper 平台，包括 Paper、Folia、Luminol。

视频内容由服务器管理还是客户端获取？

服务器管状态，客户端拉内容。 服务器负责管理频道状态，例如当前 URL、播放进度、暂停状态、播放列表和同步时间轴；客户端收到这些状态后，再自己去解析 URL、加载媒体并播放。

视频是否会占用服务器带宽？

基本不会。服务端并不负责解码视频，也不负责转发视频流，只负责播放状态管理，并发送少量用于视频同步的轻量网络包。理论上，这部分性能和带宽开销可以忽略不计。

玩家需要额外安装客户端 Mod 吗？

需要。当前仓库包含独立的客户端 Fabric Mod，频道同步、本地媒体加载和实际播放都在客户端侧完成。没有装客户端 Mod，就没有这套同步播放能力。

Part III
管理员配置手册
哪些权限给什么人、播放器公开还是私有、激活范围设多大、公共频道怎么管——服主需要知道的配置操作。
III · Admin Guide

权限一览

以下列出 MTV 的所有权限节点。面向管理员配置权限组时使用。

权限节点作用
mtv.gui打开 MTV 主菜单。
mtv.player.create创建播放器。
mtv.player.edit编辑播放器设置、外设、播放、移动、删除等。
mtv.player.edit.others编辑或删除别人创建的私有播放器。
mtv.player.teleport传送到播放器所在位置。
mtv.channel.create创建公共频道。
mtv.channel.manage.others管理别人创建的公共频道资料（名称与介绍）。
mtv.channel.control.others控制别人创建且未公开控制播放的公共频道。
mtv.admin聚合以上全部管理权限。
如果你使用旧式命令的实体选择器子命令，还需要确保玩家有 minecraft.command.selector 权限。不过面向玩家的主要流程已经全部由 GUI 覆盖，旧式命令更适合作为调试入口。

公开播放器与私有播放器

播放器的公开/私有状态决定的是谁能修改播放器的配置：

  • 私有播放器：只有创建者，或拥有 mtv.player.edit.others 的玩家可以编辑。
  • 公开播放器：任何能打开 GUI 的玩家都能看到和进入。但切回私有仍然需要创建者或 mtv.player.edit.others。

激活范围

播放器默认最大激活范围是 50 格。超出该距离后客户端会临时关闭播放器以减轻负担，回到范围内自动恢复。设为 0、off 或 none 表示无限制。

  • 室内小屏或私人屏幕 → 设置较小范围（如 10-20 格）。
  • 公共大厅主屏 → 适当加大（如 50-100 格）。
  • 极少数关键屏幕 → 设为无限制。

公共频道管理

频道页核心功能

频道页的操作在 Part II 中已经对所有用户说明。作为管理员，需要额外关注以下权限相关的内容。

频道管理权限 vs 控制权限

公共频道有两套独立的权限体系：

  • 频道管理权限：修改频道名称和介绍——频道创建者，或拥有 mtv.channel.manage.others 的玩家。
  • 频道播放控制权限：控制频道播放内容——创建者始终可控；mtv.channel.control.others 始终可控；若频道开启了"公开播放控制"，则所有玩家都可控。
管理员的常见判断场景
  • 玩家能看到频道列表 → 不代表他能改频道名和介绍。
  • 玩家能进入频道控制页 → 不代表他能修改播放内容。
  • 只有公开播放控制开启时，所有玩家才拥有播放控制权。

公共频道列表与搜索

公共频道列表支持分页浏览，支持按频道名称、频道介绍、创建者名称搜索。管理员可以通过"我的频道"快速过滤出自己创建的频道。

公开播放控制开关

这是公共频道最关键的设置：

  • 公开：所有玩家都可以控制该频道播放——适合点歌屏、互动屏。
  • 私有：只有创建者或 mtv.channel.control.others 可控制——适合公告屏、广告屏、活动主舞台。

删除公共频道

删除频道是危险操作。删除后频道本身和其状态持久化数据都会被移除。如果删除失败，通常意味着当前状态不允许删除，或该频道仍有绑定关系未解除。建议删除前先确认频道是否仍被重要播放器使用。

维护建议

  • 为公共频道建立统一命名规范，按场景、区域或功能分类。
  • 不要把所有大屏都设成无限激活范围。
  • 对重要公共屏关闭公开播放控制，把管理权只交给可信管理员。
  • 多用公共频道实现多屏同播，少用复制式维护多个独立频道。
  • 在大型建筑内，优先通过"设到玩家位置"完成摆放，再做精调。
  • 只把 mtv.channel.manage.others 和 mtv.channel.control.others 发给可信管理员。
管理员总结
MTV 的权限体系围绕三个维度展开：播放器编辑权限（公开/私有）、频道资料管理权限（名称/介绍）、频道播放控制权限（谁能改播）。理解这三层的分离后，配置权限组时就清晰了。深入的实现原理和架构参考请见下一部分。
