# MTV 使用文档与教程

本文基于当前项目实现整理，面向玩家、服主和管理人员，说明 MTV 的核心概念、日常使用方式、GUI 页面功能、命令入口和常见注意事项。

## 1. MTV 是什么

MTV 是一个以 `ItemDisplay` 为载体的媒体播放器系统。一个 MTV 播放器通常由以下几部分组成：

- 播放器本体：负责记录名字、位置、朝向、拥有者、公开/私有状态、激活范围、总音量、电源状态等。
- 屏幕外设：负责显示视频画面，可配置尺寸、亮度、偏移、旋转、填充模式、弹幕开关等。
- 扬声器外设：负责输出音频，可配置音量、范围、声道、偏移等。
- 频道：负责保存实际播放状态，例如当前媒体 URL、播放列表、播放顺序、暂停状态、播放进度等。

当前实现里，播放器和频道是分开的：

- 一个新建的播放器默认绑定自己的私有 `self` 频道。
- 播放器也可以改为绑定一个公共频道，从而与其他播放器共享同一套播放状态。
- 播放控制本质上是在控制“频道”，不是直接控制某一块屏幕。

## 2. 先理解几个核心概念

### 2.1 私有播放器与公开播放器

每个播放器都有公开/私有状态：

- 私有播放器：只有创建者，或者拥有 `mtv.player.edit.others` 权限的玩家可以编辑。
- 公开播放器：任何能打开 GUI 的玩家都能看到和进入，但切回私有时仍然需要拥有者或 `mtv.player.edit.others` 权限。

这里的“公开/私有”主要影响播放器的编辑权限，不等同于频道的播放权限。

### 2.2 私有 self 频道与公共频道

每个新建播放器默认绑定自己的 `self:<uuid>` 私有频道。

它的特点是：

- 只服务于当前播放器。
- 不出现在公共频道列表中。
- 适合单独播放、单独调试、单独摆放的显示器。

公共频道则是另一套可复用的频道：

- 由玩家手动创建。
- 可以被多个播放器绑定。
- 可以搜索、管理、编辑名称和介绍。
- 可以切换“播放权限是否公开”。

### 2.3 频道播放权限

公共频道除了“谁能管理频道资料”之外，还有“谁能控制播放”的概念。

- 频道管理权限：创建者，或者拥有 `mtv.channel.manage.others` 的玩家。
- 频道播放控制权限：
  - 创建者始终可控。
  - 拥有 `mtv.channel.control.others` 的玩家可控。
  - 如果该公共频道开启了“公开播放控制”，则所有玩家可控。

所以常见情况是：

- 你可能看得到一个公共频道，但不能改它的名字和介绍。
- 你可能能进入频道控制页，但如果频道没有开放公开控制，就不能改播放内容。

### 2.4 激活范围

播放器有一个最大激活范围，默认是 `50` 格。

- 玩家客户端超出该距离时，这个播放器会临时关闭。
- 回到范围内后会自动恢复。
- 设为 `0`、`off` 或 `none` 表示无限制。

这个设置适合控制远距离播放器的客户端负担。

### 2.5 默认新建配置

当前实现中新建一个 MTV 播放器时，会自动得到：

- 1 块屏幕 `screen_0`
- 2 个扬声器 `speaker_0` / `speaker_1`
- 默认总音量 `1.0`
- 默认最大激活范围 `50.0`
- 默认绑定私有 `self` 频道
- 默认状态为私有播放器

其中两个扬声器默认分别偏向左右声道：

- `speaker_0` 初始为 `left`
- `speaker_1` 初始为 `right`

## 3. 权限一览

项目当前定义了以下主要权限：

- `mtv.gui`：打开 MTV GUI。
- `mtv.player.create`：创建播放器。
- `mtv.player.edit`：编辑播放器设置、外设、播放、移动、删除等。
- `mtv.player.edit.others`：编辑或删除别人创建的私有播放器。
- `mtv.player.teleport`：传送到播放器位置。
- `mtv.channel.create`：创建公共频道。
- `mtv.channel.manage.others`：管理别人创建的公共频道。
- `mtv.channel.control.others`：控制别人创建且未公开控制的公共频道播放。
- `mtv.admin`：聚合以上全部管理权限。

另外，旧式命令里涉及实体选择器的子命令还依赖 `minecraft.command.selector`，但面向玩家的主要流程已经转向 GUI。

## 4. 快速上手

如果你只想尽快把一个播放器跑起来，按下面流程做即可。

### 4.1 创建一个播放器

1. 执行 `/mtv` 或 `/mtv gui` 打开主菜单。
2. 点击“创建播放器”。
3. 在聊天栏输入播放器名称。
4. 播放器会创建在你当前位置，并自动打开该播放器的控制页。

也可以直接使用命令：

```text
/mtv create <name>
```

### 4.2 给它播放内容

1. 在主菜单点击“控制最近的播放器”，或从“附近的播放器”里选中目标。
2. 进入播放器页后，打开“频道设置”。
3. 在频道页点击“设置URL”。
4. 在聊天栏输入一个支持的播放链接。
5. 新链接会被插入到当前播放之后并立即播放。

### 4.3 调整画面和音频

1. 在播放器页进入“外设管理”。
2. 点击某个屏幕，调整尺寸、偏移、旋转、亮度、填充和弹幕。
3. 点击某个扬声器，调整音量、范围、声道和偏移。

### 4.4 如果你想让多个屏幕播同一内容

1. 先创建一个公共频道。
2. 在公共频道里设置播放内容。
3. 将多个播放器绑定到这个公共频道。
4. 之后这些播放器会共享同一个频道状态和播放列表。

## 5. GUI 总体导航

当前实现以 GUI 为主入口。`/mtv` 打开的是主菜单，之后大多数控制都通过界面完成。

主菜单主要分成两组：

- 播放器侧：控制最近的播放器、浏览附近播放器、创建播放器。
- 频道侧：控制最近播放器当前绑定的频道、浏览全部公共频道、只查看我的公共频道。

通常可以按下面的思路使用：

- 想控制某一个具体屏幕设备：先找播放器，再进播放器页。
- 想控制正在播放什么：进入频道页。
- 想让多个播放器共用内容：去公共频道列表。

## 6. 主菜单教程

### 6.1 控制最近的播放器

点击“控制最近的播放器”后，系统会在你周围 `50` 格范围内寻找最近的 MTV，并打开它的播放器页。

适合：

- 你站在某台显示器旁边，想马上修改它。
- 场景里播放器数量不多，最近目标很明确。

如果附近没有播放器，会提示范围内没有 MTV。

### 6.2 附近的播放器

如果周围有很多 MTV，建议进入“附近的播放器”分页列表。

这个页面会显示：

- 播放器名称
- 距离
- 当前绑定的频道 ID

点击任意条目即可进入对应播放器页。

### 6.3 创建播放器

点击“创建播放器”后，系统会要求你在聊天栏输入名称。

注意：

- 需要 `mtv.player.create` 权限。
- 创建位置就是你当前所站位置。
- 名称不能为空。

### 6.4 频道控制

“频道控制”会直接对最近播放器当前绑定的频道进行操作。

这意味着：

- 如果最近播放器绑定的是私有 self 频道，你控制的是这个播放器自己的内容。
- 如果它绑定的是公共频道，你控制的是公共频道内容，可能影响多个播放器。

### 6.5 频道列表与我的频道

- “频道列表”显示所有公共频道。
- “我的频道”只显示你创建的公共频道。

如果你经常维护一组公共大屏，建议优先从这里进入。

## 7. 播放器页面教程

播放器页面是单台 MTV 的配置中心。

### 7.1 重命名

点击“重命名”后，在聊天栏输入新名称即可。

限制：

- 需要能编辑这个播放器。
- 对私有播放器而言，通常是创建者本人，或者拥有 `mtv.player.edit.others` 的管理者。

### 7.2 传送到位置

点击“传送到位置”会把你传送到该播放器实体所在位置。

需要权限：

- `mtv.player.teleport`

### 7.3 外设管理

点击后进入外设列表，可以查看和编辑：

- 所有屏幕
- 所有扬声器

在外设列表中还可以新增外设。

### 7.4 位置与朝向

进入后可直接微调：

- X、Y、Z 坐标
- 朝向 `Yaw`
- 俯仰 `Pitch`

支持的常见操作：

- 左键减少，右键增加。
- 潜行时步进更大。
- 可一键吸附到整数坐标。
- 可直接把实体传送到你脚下。
- 可重置朝向。

适合精调画面朝向与摆放位置。

### 7.5 激活范围

这里用于控制客户端在多远范围内保持该播放器激活。

你可以：

- `-5 / -1 / +1 / +5` 快速调节
- 点击“精确设置”后在聊天栏输入具体数值
- 直接设为“无限制”

建议：

- 室内小屏可设置小一些。
- 公共大厅主屏可适当加大。
- 只有极少数关键屏幕才建议设为无限制。

### 7.6 公开/私有切换

播放器页可以切换当前播放器的公开状态。

含义：

- 公开：更方便其他玩家查看和进入配置流。
- 私有：只有拥有者或管理者可编辑。

注意：

- 当前为公开播放器时，只有拥有者或拥有 `mtv.player.edit.others` 的玩家可以切回私有。

### 7.7 频道相关操作

播放器页底部有三项和频道相关：

- 查看当前频道 / 频道设置
- 绑定新的频道
- 切回私有频道

规则如下：

- 如果当前绑定的是公共频道，点击“查看当前频道”会进入公共频道详情页。
- 如果当前绑定的是 self 频道，点击“频道设置”会进入当前频道控制页。
- “绑定新的频道”会打开公共频道列表，让你把这个播放器绑定到某个公共频道。
- “切回私有频道”会解除公共频道绑定，重新回到独立播放模式。

### 7.8 删除播放器

点击“删除 MTV 播放器”会永久移除这个播放器和它的外设。

这是危险操作，没有撤销。

## 8. 外设管理教程

### 8.1 外设列表

外设列表会把当前播放器的所有屏幕和扬声器列出来。

你可以在这里：

- 点击屏幕进入屏幕设置
- 点击扬声器进入扬声器设置
- 点击“新增外设”创建新的屏幕或扬声器

### 8.2 新增外设

新增外设页目前提供两种选择：

- 屏幕
- 扬声器

点击后会直接追加到当前播放器，然后返回外设列表。

## 9. 屏幕设置教程

屏幕页用于精调画面显示效果。

### 9.1 尺寸

可以分别调节：

- 高度
- 宽度

操作方式：

- 增大按钮增加尺寸
- 减小按钮缩小尺寸
- 潜行时步进更大

### 9.2 亮度

亮度范围是 `0` 到 `15`。

你可以：

- 正常点击按 `1` 级增减
- 潜行点击按 `5` 级增减

### 9.3 填充模式

当前实现里主要在以下模式之间切换：

- `keep_aspect_cover`
- `fill`

适合理解为：

- 想尽量保持画面比例时，用 `keep_aspect_cover`
- 想强制铺满时，用 `fill`

### 9.4 弹幕开关

可直接切换弹幕显示开或关。

### 9.5 重置显示

“重置显示”会把和显示效果相关的基础参数恢复默认，主要用于快速回到初始可用状态。

### 9.6 偏移

可分别调整：

- X 偏移
- Y 偏移
- Z 偏移

支持：

- 左键减，右键加
- 普通步进 `0.1`
- 潜行步进 `1.0`
- 重置偏移
- 吸附偏移到整数
- 将偏移直接设到玩家脚下位置

“设到玩家位置”非常适合现场摆屏，你站到想要的位置后点一下即可。

### 9.7 旋转

当前屏幕支持围绕三个轴进行旋转微调：

- Roll(Z)
- Pitch(Y)
- Yaw(X)

支持：

- 普通点击按 `5°` 调整
- 潜行点击按 `25°` 调整
- 一键重置旋转

### 9.8 删除屏幕

如果不再需要当前屏幕，可直接删除。

这是永久移除操作。

## 10. 扬声器设置教程

扬声器页用于精调声音输出。

### 10.1 音量

每个扬声器自身有独立音量，当前实现允许在 `0.0` 到 `4.0` 范围内变化。

操作方式：

- 常规步进 `0.1`
- 潜行步进 `1.0`

### 10.2 范围

扬声器还有独立的声音传播范围。

操作方式：

- 常规步进 `1`
- 潜行步进 `5`

### 10.3 声道

可在以下模式循环切换：

- `mix`
- `left`
- `right`

推荐用法：

- 单扬声器通常用 `mix`
- 双扬声器左右摆放时分别设为 `left` / `right`

### 10.4 重置音频

会把音量、范围、声道恢复默认。

### 10.5 偏移

和屏幕类似，扬声器也支持：

- X/Y/Z 偏移微调
- 重置偏移
- 偏移吸附到整数
- 直接设到玩家脚下位置

### 10.6 删除扬声器

如果不再需要某个扬声器，可以直接删除。

## 11. 频道控制页教程

频道页是播放控制中心。无论它是某个播放器的 self 频道，还是一个公共频道，操作逻辑都基本一致。

### 11.1 你在这里控制的到底是什么

频道页控制的是该频道的运行状态，包括：

- 当前媒体 URL
- 当前播放进度
- 暂停 / 播放状态
- 播放列表
- 播放顺序

如果多个播放器绑定到同一个公共频道，那么你在频道页上的操作会同步影响所有绑定它的播放器。

### 11.2 顶部播放列表

频道页顶部会显示一个可滚动的播放列表窗口，一次显示 7 项。

你可以对单个播放项做这些操作：

- 左键：立即播放该项
- 右键：删除该项
- `Shift + 左键`：移到最前
- `Shift + 右键`：移到最后

左右箭头可以滚动查看更前或更后的列表内容。

### 11.3 列表管理按钮

频道页第二行有几类常用操作：

- 首加：把链接加到列表开头
- 下一个加：插入到当前播放项之后
- 尾加：加到列表末尾
- 清空列表：清除全部播放项并停止播放
- 播放顺序：循环切换播放模式

其中“首加”“下一个加”“尾加”都会要求你在聊天栏输入媒体链接。

### 11.4 播放顺序

当前实现会在以下模式间循环切换：

- `列表播放`
- `随机`
- `列表循环`
- `当前媒体循环`
- `播完当前停止`

如果你的应用场景不同，建议这样选：

- 展示型长播内容：`列表循环`
- 单曲或单视频重复：`当前媒体循环`
- 临时点播：`播完当前停止`

### 11.5 播放 / 暂停

频道页中部按钮用于切换播放和暂停。

前提是频道当前已有媒体内容；否则切换不会有实际效果。

### 11.6 底部遥控式控制

底部一排控制从左到右包括：

- 上一个媒体
- 后退 20 秒
- 后退 5 秒
- 设置 URL
- 前进 5 秒
- 前进 20 秒
- 下一个媒体

其中“设置 URL”的行为是：

- 要求你输入一个链接
- 将它插入到当前播放之后
- 立即播放该链接

### 11.7 权限限制

如果当前频道是公共频道，而你不是创建者，也没有频道控制权限，并且频道没有开启公开控制，那么你可以打开页面但无法真正修改播放状态。

## 12. 公共频道教程

### 12.1 公共频道列表

公共频道列表用于浏览所有可发现的公共频道。

页面支持：

- 上一页 / 下一页
- 只看我的频道 / 查看全部频道
- 输入搜索词
- 清空搜索
- 创建公共频道

每个频道条目会显示：

- 名称
- 创建者
- 简介摘要
- 当前观看人数

### 12.2 搜索

搜索支持按以下内容匹配：

- 频道名称
- 频道介绍
- 创建者名称

搜索词通过聊天栏输入。

### 12.3 从列表绑定公共频道到播放器

如果你是从某个播放器页点进“绑定新的频道”，那么此时公共频道列表会带着当前播放器上下文。

在这种情况下：

- 左键频道条目：把当前播放器绑定到这个公共频道
- 右键频道条目：进入频道管理页

绑定成功后，会直接进入该播放器对应的频道控制页。

### 12.4 创建公共频道

创建页包含三步：

1. 设置频道名称
2. 设置频道介绍
3. 点击确认创建

限制：

- 需要 `mtv.channel.create`
- 名称不能为空

创建成功后会进入该频道的详情页。

### 12.5 公共频道详情页

详情页会展示：

- 频道名称
- 频道 ID
- 创建者
- 频道介绍
- 当前观看人数

并提供以下入口：

- 频道控制
- 编辑频道名称
- 编辑频道介绍
- 切换播放权限公开/私有
- 删除频道

### 12.6 公开播放控制开关

这是公共频道最关键的设置之一。

- 公开：所有玩家都可以控制该频道播放。
- 私有：只有创建者或拥有 `mtv.channel.control.others` 的玩家可以控制。

如果你做的是公共点歌屏，建议开公开。
如果你做的是服务器公告屏、活动主舞台、广告屏，通常建议关公开。

### 12.7 删除公共频道

频道删除是危险操作。

删除后：

- 频道本身会被移除。
- 频道状态持久化数据也会删除。

如果删除失败，通常应理解为：

- 当前状态不允许删除，或
- 该频道仍有某些绑定/约束未满足

因此删除公共频道前，建议先确认它是否仍在被重要播放器使用。

## 13. 遥控器页面教程

遥控器页更适合做“站在远处快速控播”，而不是做精细配置。

### 13.1 遥控器能做什么

主要包括：

- 切换播放器电源状态
- 进入播放器设置页
- 进入频道设置页
- 调整播放器总音量
- 快速上一首 / 下一首
- 快退 / 快进
- 直接设置当前媒体 URL

### 13.2 电源与待机

遥控器左上角可切换播放器电源开关。

当前代码里的含义更接近“待机挂起 / 重新启用”，适合做远程暂时停用，而不是删除设备。

### 13.3 总音量

遥控器右侧调的是播放器总音量，不是单个扬声器音量。

当前实现里：

- 普通点击按 `0.1` 增减
- 潜行点击按 `0.25` 增减
- 总音量会被限制在 `0.0` 到 `1.0`

### 13.4 遥控播放控制

遥控器底部可以快速执行：

- 上一首
- 快退 20 秒
- 快退 5 秒
- 设置 URL
- 快进 5 秒
- 快进 20 秒
- 下一首

如果当前绑定的是一个未公开控制的公共频道，而你又没有控制权限，那么遥控器会拒绝修改播放内容。

## 14. 面向不同角色的推荐用法

### 14.1 普通玩家

推荐流程：

1. `/mtv` 打开主菜单。
2. 用“控制最近的播放器”或“附近的播放器”找到目标。
3. 在频道页里用“设置URL”“下一首”“播放列表”完成点播。

前提是频道开启了公开控制，或者你本身有对应权限。

### 14.2 建筑师 / 场景搭建者

推荐流程：

1. 先创建播放器。
2. 在播放器页微调位置与朝向。
3. 在外设页调屏幕尺寸、偏移、旋转。
4. 用“设到玩家位置”快速定位屏幕和扬声器。
5. 最后再接公共频道。

### 14.3 服主管理员

推荐流程：

1. 建立一组公共频道，按场景命名。
2. 给核心公共屏绑定对应公共频道。
3. 根据使用场景决定是否开放公开播放控制。
4. 用激活范围控制客户端负担。
5. 只把 `mtv.channel.manage.others` 和 `mtv.channel.control.others` 发给可信管理员。

## 15. 命令附录

### 15.1 玩家常用命令

```text
/mtv
/mtv gui
/mtv create <name>

```

说明：

- `/mtv`：玩家执行时直接打开主菜单。
- `/mtv gui`：显式打开主菜单。
- `/mtv create <name>`：在自己位置创建播放器。
- `/mtv help`：显示帮助。

代码里仍保留了部分 `player` / `channel` 子命令能力，例如：

- 基于实体选择器控制某个播放器
- 直接按频道 ID 控制频道
- 用命令修改播放列表、进度、音量、偏移等

但当前帮助文本已经明确提示：

因此，这些命令更适合作为兼容层、调试入口或管理员工具，而不是日常玩家主流程。

## 16. 常见场景示例

### 16.1 做一个单人私有电视

适合：家园电视、小房间屏幕、私人影院。

步骤：

1. 创建播放器。
2. 保持默认私有状态。
3. 保持绑定 self 频道。
4. 在频道页设置 URL 和播放列表。
5. 按需调节屏幕和扬声器。

优点：

- 不影响其他人
- 播放内容独立
- 配置最简单

### 16.2 做一个大厅公告大屏

适合：主城迎宾屏、活动预告屏、广告屏。

步骤：

1. 创建一个公共频道。
2. 命名为易识别的频道名，例如“主城公告屏”。
3. 关闭公开播放控制，只允许管理员改播。
4. 将一个或多个播放器绑定到该频道。
5. 统一调节屏幕朝向和大小。

优点：

- 多块屏幕可统一播控
- 玩家不能随意改播

### 16.3 做一个可多人点播的公共点歌屏

适合：酒吧、舞台、休闲区、语音活动区。

步骤：

1. 创建一个公共频道。
2. 开启公开播放控制。
3. 把对应播放器绑定到该频道。
4. 引导玩家通过频道页或遥控器进行点播。

建议：

- 用频道名称和介绍写清用途。
- 必要时安排管理员定期清理播放列表。

## 17. 常见问题

### 17.1 为什么我能看到频道，但点了没反应

最常见原因是：

- 该频道是公共频道。
- 你不是创建者。
- 你没有 `mtv.channel.control.others`。
- 频道也没有开启公开播放控制。

此时你可以查看，但不能修改播放状态。

### 17.2 为什么我不能编辑某个播放器

通常是因为：

- 该播放器是私有的。
- 你不是创建者。
- 你没有 `mtv.player.edit.others`。

### 17.3 为什么播放器离远了就不播了

检查它的“激活范围”设置。当前实现中，超过范围后客户端会临时关闭该播放器，回到范围内才恢复。

### 17.4 为什么我设置 URL 后不是替换，而是插入

当前 GUI 中“设置URL”的语义不是“清空并替换整个列表”，而是：

- 把新链接插入到当前播放之后
- 然后立即播放它

如果你想完全清空再播，先用“清空列表”，再设置 URL。

### 17.5 公开播放器和公开频道是同一个概念吗

不是。

- 公开播放器：偏向“谁能编辑这个播放器”。
- 公共频道：偏向“多个播放器是否共享同一套播放状态”。
- 公开播放控制：偏向“其他玩家能不能改这个公共频道正在播什么”。

这三者需要分开理解。

## 18. 维护建议

如果你是服务器管理员，建议遵循以下实践：

- 为公共频道建立统一命名规范，例如按场景、区域或功能分类。
- 不要把所有大屏都设成无限激活范围。
- 对真正重要的频道关闭公开播放控制。
- 多用公共频道实现“多屏同播”，少用复制粘贴式单独维护。
- 在大型建筑中优先通过“设到玩家位置”完成摆放，再做微调。

## 19. 当前实现结论摘要

为了便于快速查阅，最后给出一句话级别的结论：

- 日常入口是 `/mtv` GUI，不是命令行。
- 播放控制的核心对象是频道，不是播放器。
- 新播放器默认自带 1 屏 2 喇叭、私有 self 频道、50 格激活范围。
- 公共频道用于多播放器共播，是否允许所有人改播由“公开播放控制”决定。
- 私有播放器、公共频道、公开播放控制是三套不同的权限语义。
